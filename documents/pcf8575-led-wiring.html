PCF8575 — 8 LED Test Setup

Wiring 8 different-colour LEDs (each with its own resistor) to a PCF8575 I²C I/O expander.

The one thing to get right: the PCF8575's outputs are quasi-bidirectional — they sink current strongly but can only source ~100 µA through a weak internal pull-up. So wire each LED to sink: anode toward VCC (through its resistor), cathode to the PCF8575 pin. Writing the pin LOW (0) turns the LED ON; HIGH (1) turns it OFF.

Wiring Diagram

Each channel: +5V → resistor → LED (anode→cathode) → PCF8575 pin. The LED triangle points toward the pin, so the pin pulling LOW completes the circuit to ground inside the chip.

If your microcontroller runs at 3.3V (ESP32, Raspberry Pi…), power the chip at 5V but tie the I²C pull-ups to 3.3V. I²C is open-drain, so the lines are only ever pulled low — no level shifter needed.

Lazy option: a single 470 Ω on every channel works fine for a test rig — red runs ~6 mA, blue ~4 mA. Brightness varies slightly between colours but all light clearly, and you only need one resistor value. Keep each LED ≤ 5–10 mA so total sink current stays comfortable when many are on at once.

Quick Test (Arduino)

#include <Wire.h>
#define ADDR 0x20   // A0=A1=A2 tied to GND

void writeAll(uint16_t bits) {       // bit=0 → LED ON (pin sinks)
  Wire.beginTransmission(ADDR);
  Wire.write(bits & 0xFF);            // P00–P07
  Wire.write(bits >> 8);              // P10–P17
  Wire.endTransmission();
}

void setup() { Wire.begin(); }

void loop() {
  for (int i = 0; i < 8; i++) {       // chase one LED at a time
    writeAll(~(1 << i));              // 0 = on, so invert
    delay(150);
  }
}

Writing 0x0000 turns all 8 LEDs on; 0xFFFF turns them off.
